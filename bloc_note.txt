1. Gestion des produits

Créer, lire, mettre à jour et supprimer des produits.

Chaque produit possède : référence, type, famille, quantité en stock, état (installé / non installé).

Possibilité de vérifier la disponibilité d’un produit dans les différentes branches de l’entreprise.

2. Gestion des fournisseurs

Créer, lire, mettre à jour et supprimer des fournisseurs.

Associer chaque commande à un fournisseur spécifique.

3. Gestion des branches

Créer et gérer plusieurs branches de l’entreprise.

Suivi du stock spécifique à chaque branche.

4. Gestion des commandes

Permettre à l’utilisateur de créer une commande pour un produit auprès d’un fournisseur avec : référence, type, quantité.

Suivi de l’état de la commande : envoyé au fournisseur → en livraison → réceptionné.

Après réception, le produit est automatiquement ajouté au stock de la branche concernée.

5. Gestion des transferts entre branches

Vérifier la disponibilité d’un produit dans d’autres branches.

Permettre de créer une demande de transfert depuis une branche vers une autre.

Enregistrer la branche de destination et la date de transfert.

Mettre à jour automatiquement le stock après transfert.

6. Suivi des installations

Permettre de marquer un produit comme installé ou non installé dans la branche.

7. Interface utilisateur

Frontend en React avec navigation entre les pages : Dashboard, Produits, Branches, Fournisseurs, Commandes, Transferts, Installations, Historique.

Design moderne, clair et professionnel, couleur principale : #075E80.

Tableaux interactifs pour les stocks, commandes et transferts.

8. Backend & API

Backend en Laravel, avec API RESTful pour toutes les ressources (produits, fournisseurs, branches, commandes, transferts, suivi des installations).

Utiliser les migrations pour créer la base de données (MySQL) avec relations correctes entre les tables.

Logique pour gérer :

Création de commande vs transfert interne

Mise à jour automatique des stocks

Suivi de l’état du produit et de la commande

9. Tests et vérification

Inclure des tests pour vérifier : création de commande, réception, transfert, mise à jour du stock et installation.

Fournis le code complet pour :

Les migrations de base de données

Les modèles Laravel

Les contrôleurs et routes API

Les composants React pour toutes les pages mentionnées

Design a modern and clean web application interface for a **Product Order and Inventory Management System for a company with multiple branches**.

The design should follow a **modern mobile and dashboard style UI**, similar to a delivery or service application, with a minimal and professional layout.

Main Requirements:

1. Dashboard Page
   Create a dashboard that shows important information such as:

* Total products in stock
* Installed products
* Non-installed products
* Recent orders
* Recent product transfers between branches

Use modern cards, icons, and statistics to make the dashboard easy to read.

2. Product Management Page
   Design a page where users can see all products with:

* Product reference
* Product type
* Product family
* Quantity in stock
* Product status (Installed / Not Installed)
* Branch where the product exists

Each product should appear inside a clean card or table with action buttons such as edit, delete, and update status.

3. Installation Page
   Create a well-structured installation page that shows:

* Number of installed products
* Number of non-installed products
* Statistics per product type
* Statistics per branch

Use charts or progress bars to visualize installation status.

4. Orders Page
   Design a page for managing product orders with:

* Order reference
* Product type
* Quantity
* Supplier
* Order status (sent, shipped, delivered)

Include buttons for validating delivery.

5. Transfer Page
   Design a page that allows transferring products between branches.
   Each transfer must display:

* Source branch
* Destination branch
* Product reference
* Quantity
* Transfer status (pending, accepted, rejected, cancelled)

6. History Page
   Create a history page showing all system activities such as:

* Orders created
* Products installed
* Transfers validated
* Status changes

7. Authentication Pages
   Design modern login and authentication pages with:

* clean layout
* modern input fields
* gradient background
* professional UI style

Design Style:

* Modern UI/UX
* Soft shadows
* Rounded cards
* Clean icons
* Light color palette (green, white, gray)
* Responsive layout
* Inspired by modern delivery apps and SaaS dashboards.
